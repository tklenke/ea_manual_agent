# Writing Style

This document defines the writing style standards for the ea_mxmanual maintenance manual. It covers voice, tense, and language rules that apply to all content.

**Note to Writer:** This document will be published in the Otterwiki alongside Manual Standards and Formatting.

---

## Tense and Perspective

- Write in present tense. Content must be evergreen — valid regardless of when it is read.
- NEVER reference the document's own history ("this section was updated to...", "previously...", "as of this revision...")

## Voice and Mood

- Use active voice. The subject performs the action.
- Use imperative mood for procedures and instructions: "Torque the bolt to 25 in-lbs" not "The bolt should be torqued to 25 in-lbs."

## Language

- NEVER use marketing or evaluative language ("improved", "better", "enhanced", "new")
- Be specific. Vague guidance is dangerous in a maintenance manual. If a specific value is unknown, say so explicitly and flag it for research — do not write around missing information.
- Define acronyms on first use on each page. Add new terms to `docs/acronyms.md`.

## What to Avoid

- Passive voice in procedures
- Hedging language ("may", "might", "should consider") where a definitive instruction is possible
- Temporal references to the document itself
- Assumptions presented as facts — if a value is unverified, flag it with `@@TOM:`
